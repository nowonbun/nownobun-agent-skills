# Must

## Scope
- 이 문서는 `global_instructions.md`, `CLAUDE.md`, `AGENTS.md`, `SKILL.md` 계열 문서의 용어/강도 해석 기준에 적용한다.

## Out of Scope
- 기술 구현 상세, 아키텍처 설계, 코드 스타일 자체 규칙은 이 문서에서 직접 정의하지 않는다.

## Source of Truth
- 용어와 강도 해석의 단일 원본은 이 문서다.
- 상위 헌법 문서 작성 규칙의 단일 원본은 `constitution-rule.md`로 둔다.
- 스킬 문서 형식 규칙의 단일 원본은 `skill-create-rule.md`로 둔다.

## 강도 정의
- `반드시`, `항상`은 의무로 해석한다.
- `권장한다`는 선택 가능하나 사유를 남겨야 하는 권장으로 해석한다.
- `금지한다`, `사용하지 않는다`는 예외 없는 금지로 해석한다.
- 의무/권장/금지는 같은 항목에서 혼용하지 않는다.

## 용어 정의
- `규칙`: 수행자에게 행동을 지시하는 문장 단위다.
- `검증`: 규칙 준수 여부를 확인하는 절차다.
- `완료`: 검증 조건을 충족해 종료 가능한 상태다.
- `예외`: 기본 규칙을 제한적으로 덮어쓰는 조건이다.
- `단일 원본`: 동일 의미를 최종 판정하는 기준 문서다.

# Do NOT

## Ambiguity
- `적절히`, `필요시`, `상황에 따라`, `가능하면`을 판단 기준 없이 쓰지 않는다.
- 같은 용어를 문서마다 다른 의미로 재정의하지 않는다.

## Weakening
- 의무 표현을 일반 서술형으로 약화하지 않는다.
- 금지 표현을 권장 표현으로 완화하지 않는다.

## Duplication
- 동일 용어 정의를 여러 문서에 복제하지 않는다.
- 복제가 필요하면 정의 원문 대신 이 문서를 참조한다.

# Flow

## Authoring
1. 먼저 대상 문서에서 핵심 용어와 강도 표현을 추출한다.
2. 이 문서 정의와 충돌 여부를 확인한다.
3. 충돌 시 대상 문서를 수정하고 이 문서 기준으로 정렬한다.
4. 신규 용어가 필요하면 정의/예시/금지 오해를 함께 추가한다.

## Review
1. 의무/권장/금지 표현이 혼용되지 않았는지 확인한다.
2. 모호어 사용 여부를 확인한다.
3. 단일 원본 참조가 누락되지 않았는지 확인한다.

# Definition of Done

## Verification
- 강도 표현(의무/권장/금지)이 문장 단위로 구분된다.
- 핵심 용어 정의가 중복 없이 존재한다.
- 모호어 금지 규칙 위반 표현이 없다.
- 단일 원본 참조 문서가 명시된다.

## Monitoring
- 신규 문서 추가 시 용어 충돌 점검 체크리스트에 이 문서를 포함한다.
- 분기별 1회 이상 문서 샘플링으로 강도 표현 일관성을 점검한다.
