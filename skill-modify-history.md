# Skill Modify History

## 목적

- 사용자 기대와 실제 동작 사이에 차이가 발생했을 때 원인을 기록한다.
- 어떤 스킬, 지침, 규칙을 수정했는지 남긴다.
- 같은 문제가 반복되지 않도록 재발 방지 조치를 구조적으로 문서화한다.

## 언제 사용하나

아래 상황이면 이 스킬을 사용한다.

1. 사용자가
   - “왜 다르게 움직였냐”
   - “왜 규칙을 안 지켰냐”
   - “다음부터 이런 일이 없게 기록해라”
   라고 지적한 경우
2. 사용자 지적이 없더라도, 스킬·전역 지침·작업 규칙을 수정한 이유를 자발적으로 이력화해야 하는 경우
3. 이후 동일 문제 추적을 위해 날짜별 히스토리 문서가 필요한 경우

## 기록 위치 규칙

### 1. 사건 이력 문서

- 위치:
  - `history/skill_YYYYMMDD.md`
- 예:
  - `history/skill_20260329.md`

### 2. 규칙 문서 수정

- 실제로 수정한 문서는 함께 기록한다.
- 예:
  - `global_instructions.md`
  - `claude-review.md`
  - 관련 스킬 문서

## 필수 기록 항목

히스토리 문서에는 반드시 아래 항목을 넣는다.

1. 목적
2. 사건 요약
3. 문제 원인
4. 영향
5. 이번에 수정한 내용
6. 재발 방지 규칙
7. 체크리스트
8. 결론

## 권장 작성 형식

```md
# skill 수정 이력 - YYYY-MM-DD

## 목적

## 사건 요약

## 문제 원인

### 1. <원인 1>
### 2. <원인 2>
### 3. <원인 3>

## 영향

## 이번에 수정한 내용

### 1. <수정한 문서 1>
### 2. <수정한 문서 2>

## 재발 방지 규칙

## 체크리스트

- [ ] ...

## 결론
```

## 작성 규칙

1. 변명하지 않는다.
2. 사용자 판단이 맞으면 명확히 인정한다.
3. “왜 발생했는지”와 “무엇을 바꿨는지”를 분리해서 쓴다.
4. 추상적으로 쓰지 말고, 수정한 파일명을 명시한다.
5. 재발 방지 규칙은 실제 작업 절차로 써야 한다.
6. 체크리스트는 다음 작업 때 그대로 재사용 가능해야 한다.
7. 문서는 UTF-8로 저장한다.

## 작업 절차

1. 사용자가 지적한 기대-실제 불일치 내용을 한 줄로 요약한다.
2. 어떤 규칙을 놓쳤는지 확인한다.
3. 관련 문서(전역 지침, 스킬, 작업 규칙)를 수정한다.
4. `history/skill_YYYYMMDD.md` 파일을 만든다.
5. 원인, 수정 내용, 재발 방지 규칙을 기록한다.
6. 규칙 문서를 수정한 경우에는 교차 검토를 시도하고, 결과는 `claude-review.md` 형식으로 별도로 보고한다.

## 금지 사항

1. “실수였다”만 쓰고 끝내지 않는다.
2. 수정 파일명을 생략하지 않는다.
3. 재발 방지 규칙 없이 사건 요약만 남기지 않는다.
4. 같은 내용을 여러 문서에 중복 정의할 때는 원본 문서를 명시하지 않은 채 복제하지 않는다.

## 산출물

- `history/skill_YYYYMMDD.md`
- 수정된 규칙 문서 목록
- 필요 시 교차 검토 결과
