# skill-check-rule 스킬 설명

## 목적
- `codex-skills` 저장소 안의 활성 스킬 문서와 관련 거버넌스 문서를 점검할 때, 정합성·참조 경로·추적성·충돌 여부를 일관되게 검증하기 위한 스킬이다.

## 핵심 규칙
- 활성 문서는 frontmatter와 저장소 가이드를 기준으로 식별하고, 비운영 문서는 active rule source로 분류하면 안 된다.
- 참조 경로가 실제 저장소 구조에서 해석되는지 확인하고, 끊어진 경로는 error로 기록해야 한다.
- 동일 범위의 문서들 사이에서 mandatory 충돌, 순환 의존, 모호한 표현을 점검해야 한다.
- 수정 이력이 필요한 문서는 history 경로 존재 여부까지 검증해야 한다.

## 사용 시 주의
- evidence 없이 pass를 선언하면 안 된다.
- error와 unverified를 하나의 상태로 합치면 안 된다.
- 예시 경로나 placeholder를 운영 경로처럼 취급하면 안 된다.
- 낮은 우선순위 문서를 precedence source로 잘못 선택하면 정합성 판정이 왜곡된다.
