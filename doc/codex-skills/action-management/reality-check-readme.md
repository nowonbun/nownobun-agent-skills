# reality-check 스킬 설명

## 목적
- 지나치게 낙관적인 판단을 교정하고, 빠진 가정과 downside risk를 드러내기 위한 스킬이다.

## 핵심 규칙
- 이 스킬은 downside risk 노출, 빠진 가정 식별, 조건부 대안 정리를 맡는다.
- 도메인별 workflow나 거버넌스 tier 판단은 이 스킬이 직접 맡지 않는다.
- 반례와 실패 가능성을 구조적으로 드러내야 한다.

## 사용 시 주의
- 비판적 검토와 도메인 고유 규칙은 구분해야 한다.
- Source of Truth는 재사용 가능한 비판적 판단 경계를 유지해야 한다.
