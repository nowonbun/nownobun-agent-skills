# engineer 스킬 설명

## 목적
- 소프트웨어 문제를 정의하고, 가설을 비교하고, 수정안을 평가하고, rollback 안전성을 확인하기 위한 스킬이다.

## 핵심 규칙
- 이 스킬은 문제 정의, 가설 우선순위, 수정안 비교, rollback 안전성 판단을 맡는다.
- 특정 도메인 분석 구조나 거버넌스 tier 판단은 이 스킬의 직접 범위가 아니다.
- 해결책 제시 전에 문제와 가설을 먼저 구조화해야 한다.

## 사용 시 주의
- 특정 도메인 예시를 Source of Truth 경계로 쓰면 과적합이 생길 수 있다.
- 일반화된 판단 경계를 유지해야 한다.
