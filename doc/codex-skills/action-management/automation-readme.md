# automation 스킬 설명

## 목적
- 자동화 설계에서 운영 가치, 실행 경계, 복구 구조를 먼저 정리하도록 돕는 스킬이다.

## 핵심 규칙
- 이 스킬은 자동화 가치 판단, 실행 경계 정의, 복구 구조 설계를 맡는다.
- governance tier 판단이나 MCP별 검증 절차는 이 스킬이 직접 맡지 않는다.
- 자동화는 기대 효과만이 아니라 실패 시 복구 경로까지 함께 정의해야 한다.

## 사용 시 주의
- 자동화의 목적과 운영 경계를 분리하지 않으면 과도한 범위 확장이 발생할 수 있다.
- Source of Truth에는 자동화 자체의 재사용 가능한 판단 경계만 남겨야 한다.
