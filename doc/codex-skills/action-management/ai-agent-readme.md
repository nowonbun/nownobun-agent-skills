# ai-agent 스킬 설명

## 목적
- AI 에이전트를 설계할 때 운영 문제, 결정 경계, 도구 계약, 검증 계획을 먼저 정의하도록 돕는 스킬이다.

## 핵심 규칙
- 이 스킬은 AI 에이전트의 문제 정의, 결정 경계, 도구 계약, 검증 계획 구조를 맡는다.
- 구현 단계 협업, 리뷰 순서, 변경 보고는 `coding-assistant`의 결정 경계로 분리한다.
- 기능 나열보다 운영 문제와 실패 비용, 성공 기준을 먼저 정의해야 한다.

## 사용 시 주의
- 에이전트 설계와 실제 코드 변경 절차를 같은 문서에서 처리하면 안 된다.
- Source of Truth에는 구현 시점 규칙을 끌어오지 말고, 어떤 판단을 어디에 위임하는지만 적어야 한다.
